Setup guide

Run TK103B Tracker with Docker or from source. You need a PostGIS database — the compose file does not include one.

Prerequisites

  • PostGIS 13+ (any hosted or self-managed PostgreSQL with PostGIS extension)
  • Docker & Docker Compose (recommended), or Node.js 20+
  • TK103B tracker configured to report to your server IP on port 9000
  • Firewall: expose 9000/tcp (ingest) and 9060/tcp (web UI) as needed

Docker (recommended)

Copy the sample files from examples/ in the repository:

mkdir tk103b-tracker && cd tk103b-tracker
curl -O https://raw.githubusercontent.com/darioguarascio/tk103b-tracker/main/examples/docker-compose.yml
curl -O https://raw.githubusercontent.com/darioguarascio/tk103b-tracker/main/examples/.env.example
cp .env.example .env

Configure .env

VariableDescription
DATABASE_URLPostGIS connection string, e.g. postgres://user:pass@dbhost:5432/tk103b
AUTH_PASSWORDUI password; leave empty to disable login
AUTH_SECRETSession signing secret — generate with openssl rand -hex 32
IMAGE_TAGDocker image tag (default latest)
APP_PORTHost port for web UI (default 9060)
INGEST_PORTHost port for GPS TCP (default 9000)

Initialize database schema (once)

docker compose --profile init run --rm db-init

Start services

docker compose up -d

Open http://<your-host>:9060. Point the tracker at <your-host>:9000.

Development

git clone https://github.com/darioguarascio/tk103b-tracker.git
cd tk103b-tracker
cp .env.example .env
# edit DATABASE_URL

npm install
npm run init-db
npm run dev
  • API: localhost:3001
  • Vite dev UI: localhost:5173 (proxies /api)

Tracker configuration

TK103B devices are typically configured via SMS. Common settings (exact commands vary by firmware):

  1. Set the server IP and port to your ingest host: 9000
  2. Enable GPRS upload interval (e.g. every 30s while moving)
  3. Verify the device sends imei:… prefixed packets — this is what the ingest server parses

On connect the server responds with LOAD or ON handshakes as required by the protocol.

Database schema

Two tables: tracker (one row per IMEI) and car (position/event log with PostGIS geometry(Point, 4326)). Schema is in migrations/001_schema.sql and applied by npm run init-db.

Releases & Docker images

Pushing a version tag builds and publishes images to Docker Hub:

git tag v1.0.0
git push origin v1.0.0

Images: darioguarascio/tk103b-tracker and darioguarascio/tk103b-tracker-ingest. Pin IMAGE_TAG=v1.0.0 in your .env for a specific release.

Legacy data migration

If you have data in an old single-table car schema without tracker_id:

SOURCE=postgres://old-host/db TARGET=postgres://new-host/tk103b npm run migrate-legacy

Troubleshooting

  • No positions appearing — check ingest logs (docker compose logs ingest), firewall on port 9000, and that DATABASE_URL is reachable from containers
  • UI shows login loop — set a stable AUTH_SECRET; without it, sessions invalidate on every app restart
  • Empty map — run db-init, confirm tracker IMEI appears in GET /api/trackers
  • Health check — curl http://localhost:9060/api/health should return {"ok":true}
